Rebuild this entire project from scratch as a quantitative 
trading screener web application. This is based on my first 
project's exact strategy. No Telegram, no bot — web dashboard 
only.

=== CORE CONCEPT ===
Detect institutional liquidity sweeps across multiple asset 
classes, score each asset using multiple indicators, rank 
them by probability and display on a live web dashboard.

=== SWEEP FORMULA (most important) ===
sweep_percent = ((candle_high - candle_low) / daily_ATR) × 100
Flag as sweep if sweep_percent > 20%
Bullish confirmed: candle is bullish AND next candle closes 
above sweep candle high
Bearish confirmed: candle is bearish AND next candle closes 
below sweep candle low

=== EXACT INDICATOR FORMULAS ===
RSI (period 14):
delta = close.diff()
gains = delta.where(delta>0, 0).rolling(14).mean()
losses = (-delta.where(delta<0, 0)).rolling(14).mean()
rsi = 100 - (100 / (1 + gains/losses))

ATR (period 14):
tr = max(high-low, abs(high-prev_close), abs(low-prev_close))
atr = tr.rolling(14).mean()
daily_atr = fetch from daily candles specifically

EMA (period 20):
ema = close.ewm(span=20, adjust=False).mean()

PIVOT POINTS (from previous day's candle):
pp = (high + low + close) / 3
r1 = (2 × pp) - low
r2 = pp + (high - low)
s1 = (2 × pp) - high
s2 = pp - (high - low)

FIBONACCI (from recent 20-candle swing high/low):
Retracements: 0.236, 0.382, 0.500, 0.618, 0.786
Extensions: 1.272, 1.414, 1.618, 2.000, 2.618
retrace_level = swing_high - (diff × level)
ext_level = swing_high + (diff × (level - 1))

MONTHLY DEMAND ZONES:
Get last 6 monthly candle closes from daily data
Flag if current price within 1% of any monthly close

VWAP:
typical_price = (high + low + close) / 3
vwap = (typical_price × volume).cumsum() / volume.cumsum()
Use as sentiment context only

=== SCORING SYSTEM ===
Each asset gets a buy_score and sell_score.
Weights:
- Liquidity sweep confirmed: 30 points
- RSI oversold (<30) → buy, overbought (>70) → sell: 20 points
- Price above EMA20 → buy, below → sell: 15 points
- Price near S1/S2 → buy, near R1/R2 → sell: 25 points
- Price at Fibonacci retracement: 20 points
- Confirmation candle: 30 points
- Near monthly demand zone: 15 points

Minimum score to appear in results: 60 points
Score difference needed: buy_score must exceed sell_score 
by at least 30 to signal BUY, vice versa for SELL

=== MARKET CONTEXT (adjusts weights dynamically) ===
CRISIS: current ATR > average ATR × 2.0
  → reduce all weights by 30%, VWAP becomes irrelevant
TRENDING: abs(EMA20 - EMA50) / EMA50 > 2%
  → increase sweep weight by 20%, increase EMA weight by 20%
NORMAL: everything else
  → use base weights as above

=== RISK LEVELS ===
Stop loss: entry ± (daily_ATR × 2.0)
Take profit: 1.618 Fibonacci extension level
  fallback: entry ± (daily_ATR × 3.0)

=== MARKOV REGIME MODEL ===
6 market states: BULLISH, BEARISH, OVERBOUGHT, OVERSOLD, 
HIGH_VOL, SIDEWAYS

State classification per candle:
- RSI > 70 → OVERBOUGHT
- RSI < 30 → OVERSOLD  
- ATR > avg_ATR × 1.5 → HIGH_VOL
- close > EMA20 and 40 < RSI < 60 → BULLISH
- close < EMA20 and 40 < RSI < 60 → BEARISH
- else → SIDEWAYS

Build transition matrix from last 30 candles of states.
Matrix updates with every new scan (time-inhomogeneous).
If Markov probability of next state > 0.55 → boost 
signal confidence by 20%
If < 0.40 → reduce signal confidence by 20%

=== DATA FETCHING ===
15min candles: fetch last 100 candles per asset
Daily candles: fetch last 90 days per asset
Crypto: use Binance public endpoint
  https://api.binance.com/api/v3/klines
  params: symbol, interval=15m, limit=100
Stocks/Forex/Indices: use yfinance
Cache data per symbol for 15 minutes to avoid 
hitting API rate limits

=== SCAN SCHEDULE ===

  → scan all assets
